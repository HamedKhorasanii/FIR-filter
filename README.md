# FIR Filter in VHDL

This project implements a **4-tap FIR (Finite Impulse Response) low-pass filter** in VHDL along with a complete testbench for simulation and verification.

The design filters a noisy sine-wave input signal and produces a smoother output signal.

---

## Project Files

| File             | Description                                  |
| ---------------- | -------------------------------------------- |
| `fir_filter.vhd` | FIR filter implementation                    |
| `fir_tb.vhd`     | Testbench for simulation                     |
| `output.txt`     | Simulation output generated by the testbench |

---

## FIR Filter Overview

The FIR filter is implemented using the following equation:

[
y[n] = b_0x[n] + b_1x[n-1] + b_2x[n-2] + b_3x[n-3]
]

### Filter Coefficients

| Coefficient | Value |
| ----------- | ----- |
| `b0`        | 5     |
| `b1`        | 59    |
| `b2`        | 59    |
| `b3`        | 5     |

These coefficients are scaled for fixed-point arithmetic.

## FIR Filter Architecture

### Inputs

| Signal | Type                 | Description  |
| ------ | -------------------- | ------------ |
| `clk`  | `STD_LOGIC`          | System clock |
| `rst`  | `STD_LOGIC`          | Reset signal |
| `x_in` | `SIGNED(7 downto 0)` | Input signal |

### Output

| Signal  | Type                 | Description     |
| ------- | -------------------- | --------------- |
| `y_out` | `SIGNED(9 downto 0)` | Filtered output |

---

## Internal Operation

The filter performs the following steps:

1. Stores previous input samples using shift registers
2. Multiplies each sample by its coefficient
3. Adds all multiplication results
4. Scales the final sum
5. Produces filtered output

---

## Testbench Description

The testbench:

* Generates a clock signal
* Applies reset
* Creates a sine-wave lookup table
* Adds pseudo-random noise
* Sends noisy data into the FIR filter
* Stores input/output pairs into `output.txt`

### Noise Characteristics

* Random noise range: `-30` to `+30`
* Input saturation range: `-128` to `127`

---

## Simulation Flow

### 1. Compile the Design

Using ModelSim:

```bash
vcom fir_filter.vhd
vcom fir_tb.vhd
```

### 2. Run Simulation

```bash
vsim fir_tb
run -all
```

### 3. Check Results

After simulation, open:

```text
output.txt
```

Each line contains:

```text
<input_signal> <filtered_output>
```

Example:

```text
12 4
18 10
25 17
```

## Expected Result

The output signal should appear smoother than the noisy input signal because the FIR filter attenuates high-frequency noise components.
